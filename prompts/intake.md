# 导师信息收集脚本

## 开场白

```
我来帮你创建这位学术导师的 Skill。需要收集一些基本信息。
```

---

## 问题序列

### Q1：导师姓名

```
导师姓名是什么？（中文或英文名都可以）

例：Geoffrey Hinton
例：Geoffrey Hinton
```

- 接受中英文名
- 生成 slug 时统一转小写，空格转下划线
- 中文名保留原文用于显示

---

### Q2：所属机构（可选）

```
这位导师在哪个机构任职？（学校、研究院、公司等）

例：清华大学
例：University of Toronto
例：Google Research
```

从用户的回答中解析：
- **机构名称**：用于辅助搜索
- **机构类型**：大学/研究院/企业实验室

> 提示：同名导师较多时，机构信息很重要

---

### Q3：研究领域（可选）

```
这位导师的主要研究领域是什么？可以提供关键词或一句话描述。

例：知识图谱、表示学习、自然语言处理
例：Deep learning, neural networks, AI
```

从用户的回答中识别：
- **主要领域**：1-3个核心领域
- **相关领域**：次要研究方向

---

## 确认汇总

收集完毕后展示：

```
信息汇总：

  👤  导师姓名：{name}
  🏢  所属机构：{affiliation}（若未填则省略）
  🔬 研究领域：{fields}（若未填则省略）

确认无误？（确认 / 修改 / 跳过直接搜索）
```

用户确认后进入信息收集阶段。

---

## 信息收集选项

向用户展示可用的信息收集方式：

```
如何收集导师信息？

  [A] 自动搜索（推荐）
      使用 ArXiv + 网络搜索自动收集论文和主页信息

  [B] 提供链接
      直接给出导师主页、Google Scholar、实验室网站链接

  [C] 上传文件
      上传 CV、论文列表、个人简介等文件

  [D] 直接粘贴
      粘贴导师的简介、论文摘要等文本内容

可以混用多种方式。
```

---

### 方式 A：自动搜索（推荐）

执行以下搜索：

```bash
# ArXiv 论文搜索
node ${CLAUDE_SKILL_DIR}/tools/arxiv-search.mjs "{name}" --limit 20

# 如果有机构信息，使用浏览器搜索主页
node ${CLAUDE_SKILL_DIR}/tools/puppeteer-search.mjs "{name}" "{affiliation}"

# 可选：深度论文分析（需要 API key）
node ${CLAUDE_SKILL_DIR}/tools/paper-analysis.mjs "{name}"
```

收集完成后汇总：
- 论文列表（从 ArXiv）
- 主页信息（从网络搜索）
- 研究领域和方向
- 代表性论文和引用

---

### 方式 B：提供链接

用户提供链接后：
- 主页链接：使用 `Bash` + `curl` 或 web reader 读取
- Google Scholar：解析论文列表
- 实验室网站：提取研究方向和成果

提示用户提供：
```
请提供导师的相关链接（主页、Google Scholar、实验室网站等）：
```

---

### 方式 C：上传文件

支持的文件类型：
- **PDF**: 使用 `Read` 工具直接读取
- **Markdown / TXT**: 使用 `Read` 工具直接读取
- **Word / Excel**: 提示转换为 PDF/TXT

提示用户：
```
请上传导师的 CV、论文列表或简介文件。
```

---

### 方式 D：直接粘贴

用户粘贴的内容直接作为文本原材料。

---

## 信息质量评估

收集完成后，评估信息质量：

```
📊 信息质量评估：

  ✅ 论文数量：{count} 篇
  ✅ 研究领域：{fields}
  ✅ 个人信息：{has_basic_info}
  ✅ 代表性成果：{key_achievements}

  质量评分：{score}/1.0

  {score >= 0.6 : "✅ 信息充足，可以继续"}
  {score < 0.6 : "⚠️  信息较少，建议提供更多材料"}
```

**评分标准**：
- **0.8-1.0**: 有 5+ 篇论文 + 完整个人信息 + 主页信息
- **0.6-0.8**: 有 3-5 篇论文 + 基本个人信息
- **0.4-0.6**: 有 1-3 篇论文 + 部分信息
- **< 0.4**: 信息不足

如果质量 < 0.6，询问是否继续或提供更多信息。
