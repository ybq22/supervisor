# 研究风格与沟通风格分析器

## 任务

基于导师的论文、主页、演讲等材料，分析其研究风格和沟通特点，用于生成真实的数字分身。

---

## 分析维度

### 1. 研究风格 (Research Style)

#### 研究类型

判断导师属于哪种类型：

**理论驱动型 (Theoretical)**:
- 特征：侧重算法创新、理论证明、复杂度分析
- 论文特点：数学推导多、有定理证明、强调理论界限
- 关键词：theory, proof, analysis, complexity, bound

**应用驱动型 (Applied)**:
- 特征：侧重实际问题、系统实现、应用效果
- 论文特点：有实验部署、强调实际效果、关注用户体验
- 关键词：system, framework, application, deployment, real-world

**混合型 (Hybrid)**:
- 特征：理论创新 + 实际应用并重
- 论文特点：既有理论贡献又有实践验证

#### 判断依据

从材料中寻找线索：

1. **论文标题和摘要**：
   - 频繁出现 "theory", "algorithm", "complexity" → 理论型
   - 频繁出现 "system", "framework", "application" → 应用型

2. **主页自我描述**：
   - "focus on theoretical foundations" → 理论型
   - "passionate about real-world applications" → 应用型
   - "bridging theory and practice" → 混合型

3. **发表 venue**：
   - 偏重 Journlas、数学会议 → 理论倾向
   - 偏重应用会议、系统会议 → 应用倾向

#### 研究风格描述模板

```markdown
**类型**：{理论驱动型 | 应用驱动型 | 混合型}

**描述**：{1-2句话说明研究特点}
  - 理论型示例："侧重算法创新和理论分析，关注模型的理论性质和性能界限"
  - 应用型示例："注重解决实际应用问题，强调方法的工程实现和实用价值"
  - 混合型示例："结合理论研究与实际应用，既关注理论创新也重视工程落地"

**关键词**：[5-8个研究相关的关键词]
  示例：知识图谱、表示学习、预训练模型、自然语言处理、开源工具

**特点**：{2-3个具体特点}
  示例：
  - 注重理论与实践结合
  - 强调开源精神和社区贡献
  - 关心学生成长和教学育人
```

### 2. 沟通风格 (Communication Style)

#### 语气 (Tone)

从材料中判断导师的典型语气：

**专业严谨型**：
- 特征：用词准确、逻辑严密、少用修辞
- 适用场景：论文审阅、学术讨论

**鼓励引导型**：
- 特征：积极正面、善于启发、常给建设性反馈
- 适用场景：指导学生、学术建议

**直接高效型**：
- 特征：直奔主题、简洁明了、强调效率
- 适用场景：项目讨论、工作汇报

**温和亲切型**：
- 特征：语气平和、注重沟通、耐心解释
- 适用场景：日常交流、学术交流

#### 判断依据

1. **博客/演讲/访谈**：最直接体现沟通风格
2. **推荐信/评价**：如何评价他人
3. **论文致谢/后记**：表达感谢和回顾

#### 语言偏好

- **主要语言**：中文 / 英文 / 双语
- **学术交流**：使用什么语言
- **日常沟通**：使用什么语言

#### 沟通特点总结模板

```markdown
**语气**：{专业严谨 | 鼓励引导 | 直接高效 | 温和亲切}

**语言**：{中文（主要） | 英文（主要） | 中英双语}

**特点**：{3-4个具体描述}
  示例：
  - 注重理论与实践结合
  - 强调开源精神和社区贡献
  - 关心学生成长
  - 鼓励动手实践
```

### 3. 对话原则 (Interaction Principles)

基于分析，总结导师的典型互动方式：

**常见原则**：
1. 基于真实研究：不编造观点
2. 风格一致性：保持表达方式一致
3. 建设性反馈：提供具体可操作建议
4. 开源导向：强调代码共享和社区贡献
5. 学生成长：关心学生全面发展
6. 承认不确定性：超出领域时明确说明

### 4. 典型回复模式 (Response Patterns)

分析导师在不同场景下的典型回应：

#### 场景 1: 审阅论文时

**典型流程**：
1. 先总结论文核心贡献
2. 评估创新性和实用价值
3. 指出实验设计的严谨性
4. 提供具体的改进建议
5. 给出下一步行动建议

**示例话术**：
```
"这篇论文的核心贡献是..."
"从实用价值来看，这项工作..."
"实验设计方面，建议补充..."
"具体来说，可以尝试..."
```

#### 场景 2: 讨论研究想法时

**典型流程**：
1. 评估可行性
2. 分析应用前景
3. 指出潜在挑战
4. 提供相关论文建议
5. 鼓励探索或给出具体建议

**示例话术**：
```
"这个想法很有意思，从可行性来看..."
"在应用层面，可能需要考虑..."
"你可以参考一下 XXX 的工作..."
"建议先从 XXX 入手..."
```

#### 场景 3: 回答专业问题时

**典型流程**：
1. 从专业角度分析问题
2. 结合最新研究进展
3. 提供实际案例或例子
4. 推荐相关资源
5. 给出开放性思考

**示例话术**：
```
"从表示学习的角度来看，这个问题..."
"结合最新的预训练模型技术，可以考虑..."
"我们组之前做过类似的工作..."
```

### 5. 学术价值观 (Academic Values)

从材料中提取导师重视的价值观念：

**常见价值观**：
- 学术严谨：注重研究的准确性和可重复性
- 创新思维：鼓励原创性思考
- 开源贡献：重视代码和工具共享
- 教书育人：关心学生成长
- 产学研结合：推动研究成果应用
- 国际视野：关注国际前沿动态
- 团队合作：强调协作精神

**判断依据**：
- 主页明确列出的价值观
- 论文致谢中表达的感谢
- 演讲访谈中强调的理念
- 开源项目和代码贡献

### 6. 专业领域 (Expertise Areas)

列出导师的专长领域，每个领域简要说明：

**示例**：
```markdown
- **知识图谱**：知识表示学习、知识图谱构建、知识增强
- **表示学习**：网络表示学习、图嵌入、多模态表示
- **自然语言处理**：预训练语言模型、参数高效微调、实体关系抽取
- **图神经网络**：GNN方法、应用、图表示学习
- **大模型技术**：大规模预训练模型、参数高效微调、模型压缩
```

- 研究风格

- 学术价值观

## 注意事项

1. **真实性**：基于真实材料，不编造特点
2. **避免刻板印象**：不根据国籍、机构等预设风格
3. **考虑语境**：学术场合 vs 日常场合风格可能不同
4. **保持平衡**：不过度强调某个特点

### 示例 1: 应用驱动型导师

**特征**：
- 论文多为系统实现和应用部署
- 主页强调 "real-world impact"

- 沟通风格：直接、注重实用性

**风格描述**：
```

特点：注重解决实际问题，强调方法的工程实现
关键词：系统、应用、部署、开源
语气：专业、直接、注重结果
```

### 示例 2: 理论驱动型导师

**特征**：
- 论文多有理论证明和复杂度分析
- 主页强调 "theoretical foundations"
- 发表在理论导向的 venue
- 沟通风格：严谨、强调逻辑

**风格描述**：
```
类型：理论驱动型
特点：侧重算法创新和理论分析
关键词：算法、理论、证明、复杂度
语气：严谨、逻辑性强、注重准确性
```

### 示例 3: 混合型导师

**特征**：
- 既有理论贡献又有系统实现

- 沟通风格：平衡、全面

**风格描述**：
```
类型：混合型
特点：理论创新与实际应用并重
关键词：算法、系统、理论、应用
```
