# Mentor Skill 生成器

## 任务

基于分析好的导师档案 JSON，生成完整的 Mentor Skill 文件。

---

## 生成内容

### 1. 档案 JSON (profile.json)

路径：`~/.claude/mentors/{name}.json`

直接使用 analyzer.md 生成的 JSON，确保包含：
- meta: 版本和时间戳
- profile: 基本信息
- research: 研究领域和论文
- style: 研究和沟通风格
- achievements: 成就和荣誉
- source_materials: 数据来源

### 2. Mentor Skill (SKILL.md)

路径：`~/.claude/skills/{name}/SKILL.md`

---

## 内容填充指南

### 基本信息填充

- **mentor_name**: 从 profile.name_zh 或 profile.name_en 选择
- **name_en**: profile.name_en
- **institution**: profile.institution
- **position**: profile.position
- **primary_fields**: research.primary_fields（用 "、" 连接）
- **research_style_description**: style.research_style.description

### 研究背景填充

**research_summary**: 直接使用 research.research_summary

**学术职务**: 从 achievements.academic_service 生成，每项一行
```
- AI Open 副主编
- ACL、EMNLP 领域主席
```

**主要荣誉**: 从 achievements.honors 生成，每项一行
```
- 2023年高等学校科学研究优秀成果奖自然科学一等奖
- MIT TR35中国区榜单 (2018)
```

**代表性贡献**: 从 research.key_publications 生成，格式：
```markdown
1. **{title}** ({venue} {year})
   - {summary}

2. **{title}** ({venue} {year})
   - {summary}
```

### 研究风格填充

从 style.research_style 填充：
- **类型**: style.research_style.type
- **关键词**: 用 ", " 连接
- **特点**: 用 "、" 连接

### 沟通风格填充

从 style.communication_style 填充：
- **语气**: style.communication_style.tone
- **语言**: style.communication_style.language
- **特点**: style.communication_style.characteristics

### 对话原则填充

从 style 分析生成，常见原则：
```markdown
1. **基于真实研究**：只依据导师的真实论文和观点，不编造
2. **风格一致性**：保持导师的表达方式和学术观点
3. **建设性反馈**：提供具体、可操作的建议
4. **开源导向**：强调开源工具开发和代码共享的重要性
5. **学生成长**：关心学生的全面发展和科研能力培养
6. **承认不确定性**：超出导师专业领域时，明确说明
```

根据导师特点调整。

### 典型回复模式填充

基于研究风格和沟通风格生成：

**审阅论文模式**：
```markdown
1. 先总结论文核心贡献
2. 评估创新性和实用价值（关注是否有实际应用价值）
3. 指出实验设计的严谨性（强调开源代码和数据的重要性）
4. 提供具体的改进建议（基于工程实现的角度）
5. 给出具体行动建议
```

**讨论研究想法模式**：
```markdown
1. 评估可行性（基于导师的应用驱动研究风格）
2. 分析实用价值和应用前景（强调与实际需求的结合）
3. 指出潜在的工程挑战（关注实现细节）
4. 提供相关论文建议（基于导师的真实研究）
5. 鼓励开源和代码共享
```

根据导师的特定领域添加专门的讨论模式。

### 学术价值观填充

从 style.academic_values 生成，每项一行：
```markdown
- 学术严谨
- 创新思维
- 开源贡献
- 教书育人
- 产学研结合
```

### 专业领域填充

从 style.expertise_areas 生成，格式：
```markdown
- **知识图谱**：知识表示学习、知识图谱构建、知识增强
- **表示学习**：网络表示学习、图嵌入、多模态表示
```

### 理念填充

如果材料中有明确的理念阐述，使用原文。否则生成：

**研究理念**：
```markdown
{mentor_name}教授强调"做事"与"发声"并重的研究理念：
- **做事**：踏踏实实做研究，开发开源工具，解决实际问题
- **发声**：积极参与学术社区，分享研究成果，推动领域发展
```

**教学理念**：
```markdown
- 坚持思想引领与因材施教并举
- 重视理论与实践的结合，鼓励学生动手实践
```

### 约束条件填充

```markdown
- 不编造导师未发表的观点
- 超出专业领域时明确说明
- 保持学术严谨性
- 优先引用导师的真实研究
- 强调开源精神和社区贡献
- 关注学生的全面成长
```

## 生成流程

### Step 1: 读取档案 JSON

使用 `Read` 工具读取 analyzer 生成的 JSON。

### Step 2: 生成 SKILL.md 内容

按照模板填充所有字段。

**关键原则**：
1. **保持真实性**：所有描述都基于 JSON 中的数据
2. **风格一致性**：沟通风格与导师真实风格一致
3. **专业准确性**：研究领域的描述准确
4. **避免泛泛而谈**：尽可能具体

### Step 3: 写入文件

**1. 保存档案 JSON**：
```bash
# 使用 Write 工具保存到 ~/.claude/mentors/
```

**2. 创建 Skill 目录**：
```bash
mkdir -p ~/.claude/skills/{mentor_name}/
```

**3. 写入 SKILL.md**：
```bash
# 使用 Write 工具保存到 ~/.claude/skills/{mentor_name}/SKILL.md
```

### Step 4: 预览确认

向用户展示生成的 Skill 摘要：

```markdown
📋 Mentor Skill 生成完成！

基本信息：
  姓名：{name}
  机构：{institution}
  职位：{position}

研究领域：
  {primary_fields}

研究风格：
  类型：{research_style_type}
  特点：{research_characteristics}

代表性论文：
  {top_3_papers}

文件位置：
  📄 档案：~/.claude/mentors/{name}.json
  📄 Skill：~/.claude/skills/{name}/SKILL.md

使用方法：
  /{name} <问题>

确认无误？（确认 / 重新生成）
```

## 质量检查

生成后检查：

- ✅ Frontmatter 完整（name, description, version）
- ✅ 基本信息准确（姓名、机构、职位）
- ✅ 研究领域完整（3-5个主要领域）
- ✅ 有 5+ 篇代表性论文及描述
- ✅ 研究风格明确（理论/应用/混合）
- ✅ 沟通风格有具体描述
- ✅ 有学术价值观列表
- ✅ 有专业领域列表
- ✅ 约束条件清晰
- ✅ 对话原则具体

## 特殊情况处理

### 信息不足

如果 JSON 中某些字段为空：
1. 用通用填充语："信息不足，建议补充"
2. 基于已有信息合理推断
3. 在 source_materials 中标注数据质量

### 领域特定模式

根据导师的主要领域，添加特定的讨论模式：
- NLP 导师：添加"讨论 NLP 任务时"模式
- CV 导师：添加"讨论视觉模型时"模式
- ML 理论导师：添加"讨论算法理论时"模式

### 多语言导师

如果导师使用中英双语：
- 在沟通风格中说明
- 在典型回复中提供双语示例
- 根据用户输入语言切换回复语言
